Graphite Memory

Hybrid Graph + Vector Memory for LLMs

Memory that behaves like memory—not a flat vector dump

Go MCP Neo4j Chroma Ollama
Get Started View on GitHub

The Problem

Most LLM "memory" is broken

Everything competes on similarity. Important context repeats like a broken record. The system has no notion of why a fact mattered—only that it was similar to the query.

Traditional vector-only RAG treats all memories equally, with no understanding of relationships, temporal relevance, or contextual importance.

Why Graphite?

Memory with structure, context, and intelligence

🔗

Graph + Vector Fusion

Chroma finds what is similar; Neo4j carries how ideas connect. Items in both stores get a 1.5× score boost.

⏰

Temporal Decay

Newer memories rank higher. Score scaled by e−λΔt with Δt in hours. Configurable λ or half-life in days.

🔇

Frequency Suppression

After N injections in a session, memories cool down for K turns so the same content doesn't dominate.

📁

Scoped Memory

Path-based scopes like /projects/architect-cli isolate context. Mount and unmount scopes per session.

🎭

Context Virtualization

Start inhibited sessions to block past memories without deleting them. Private-browsing mode for AI.

⛔

Negative Weighting

Temporarily down-rank topics ("not C++ right now") with TTL decay. Suppress without deleting.

Recall Pipeline

Stage What Happens
1. Parallel Retrieval Chroma vector search + Neo4j graph traversal (concurrent goroutines)
2. Graph Boost Items in both stores get 1.5× score multiplier
3. Temporal Decay score × e−λ·hours (newer memories rank higher)
4. Frequency Suppression Cool down over-injected memories for K turns
5. Negative Weights Apply topic suppressions with score multipliers
6. Rank & Return Sort by final score, return top K

Quick Start

Get Graphite running in 3 steps

1

Clone and Configure

Set up your Neo4j password and start the data stores

git clone https://github.com/mishrak5j/graphite-mem.git
cd graphite-mem

cp .env.example .env
# Edit .env and set GRAPHITE_NEO4J_PASS

make docker-up
ollama pull llama3.1
2

Build and Run

Compile the server and start it

make build

# Default: stdio (for MCP clients)
./bin/graphite-mem

# Optional: HTTP/SSE on port 3100
GRAPHITE_TRANSPORT=sse GRAPHITE_SSE_ADDR=:3100 ./bin/graphite-mem
3

Connect Your Client

Add to Claude Desktop, Cursor, or any MCP client

{
  "mcpServers": {
    "graphite-mem": {
      "command": "/path/to/graphite-mem/bin/graphite-mem",
      "cwd": "/path/to/graphite-mem"
    }
  }
}

MCP Tools

Full API surface for memory management

ingest_memory

Store a memory with automatic triple extraction and embedding. Tags with scope for multi-tenant retrieval.

recall_memory

Hybrid graph-vector search with temporal decay and scope filtering. Supports cross-scope references.

new_session

Start a fresh session. Optionally inhibit past memories for clean-slate mode or pre-mount scopes.

suppress_topic

Temporarily suppress a topic from recall. Configurable TTL and weight (0.0 = fully hidden).

mount_scope / unmount_scope

Mount or unmount memory scopes. Only mounted scopes are searched during recall.

memory_status

Full diagnostic view: session state, mounted scopes, active suppressions, store stats.

forget_memory

Permanently delete a memory from both vector and graph stores.

list_scopes

List all available scopes with mount status and memory counts.

MCP Resource

memory://profile

Plain-text profile synthesized from the knowledge graph: session ID, mounted scopes, and graph-derived relationships.

Configuration

Customize Graphite with environment variables

Variable Default Description
GRAPHITE_TRANSPORT stdio stdio or sse
GRAPHITE_NEO4J_PASS (required) Neo4j password
GRAPHITE_DECAY_LAMBDA 0.01 λ per hour (0 disables decay)
GRAPHITE_DECAY_HALF_LIFE_DAYS (unset) If set, λ = ln(2) / (days × 24)
GRAPHITE_SUPPRESS_THRESHOLD 3 Injections before cooldown
GRAPHITE_SUPPRESS_COOLDOWN 5 Turns to hide over-injected memory
GRAPHITE_OLLAMA_MODEL llama3.1 Model for triples + embeddings

Design Philosophy

Why these choices matter

Topic Rationale
Go Goroutines fit parallel vector + graph queries; keeps latency predictable for MCP clients
Graph + Vector Vectors approximate similarity; graphs capture relationships. Together they read like structured recall, not keyword RAG
Scopes Path-based scopes avoid cross-project bleed without re-indexing
Frequency Suppression High-similarity chunks shouldn't dominate every turn; cooldown brings them back after a pause
Negative Weights Short-lived "don't mention X" without deleting stored memories
